## 제품 정의

NINETY-NINE VINTAGE는 외부 판매자가 입점하는 마켓플레이스가 아니다. 하나의 사업체가 여러 매장, 운영자, 직원을 통합해 상품 판매와 중앙 출고를 수행하는 멀티스토어 빈티지 커머스 시스템이다.

초기 조직은 시스템 관리자 1명과 매장 운영자 2명으로 시작한다. 향후 매장과 각 운영자 소속 직원을 추가할 수 있어야 한다.

## 변경할 수 없는 운영 원칙

1. 상품은 사업체 소유의 여러 매장에 귀속된다.
2. 외부 셀러 입점, 입점 심사, 상품 승인자, 제3자 품질 보증 제도를 제품에 도입하지 않는다.
3. 매장 운영자와 명시적 권한을 받은 직원은 자기 매장 상품을 등록하고 즉시 공개한다. 상품 등급·하자·실측·사진은 객관적인 상품 상태 정보이며 제3자 확인이나 품질 보증을 뜻하지 않는다.
4. 고객의 장바구니, 주문, 결제는 매장별로 분할하지 않는다.
5. 여러 매장 상품도 통합 주문 1건과 공용 계좌 입금 1회로 처리한다.
6. 권한 있는 운영자는 통합 입금을 공동 조회하고 한 명이 전체 주문의 입금을 확인할 수 있다.
7. 입금 확인 이후의 상품 준비·이동 책임은 원등록 매장별로 분리한다. 매장 A는 중앙 출고지 B로 인계하고, 매장 B 상품도 현장 인계와 중앙 수령을 별도 기록한다.
8. 각 매장은 고객에게 직접 발송하지 않는다. B가 유일한 중앙 출고지이며, 실제 발송은 B에서만 수행한다.
9. 고객은 결제 완료 후 보관 상품을 주문 경계와 무관하게 선택해 하나의 배송 요청으로 묶을 수 있다. 한 배송 요청은 B에서 하나의 합포장과 하나의 송장으로 발송한다.
10. 한 주문의 상품은 여러 배송 요청으로 나누어 보낼 수 있으나, 한 상품은 진행 중인 배송 요청 하나에만 포함할 수 있고 발송 완료 후 재신청할 수 없다.
11. 고객 보관 상품의 물리적 기준 위치는 B 중앙 출고지다. 결제 완료 사실과 실제 위치·입고 사실은 같은 상태로 취급하지 않는다.
12. 내부에서는 원등록 매장별 작업·매출, 상품별 물류 상태, 배송 요청을 분리하고 고객에게는 결제와 배송 요청 하나씩에 집중할 수 있는 화면을 제공한다.

## 결제 원칙

- 현재 활성 결제 수단은 수동 계좌이체뿐이다.
- PortOne은 고객 및 운영자의 정상 업무 흐름에서 실행되지 않아야 한다.
- PortOne 코드는 삭제하지 않고 향후 재활성화할 수 있는 격리된 결제 어댑터 또는 `legacy/future integration`으로 보존한다.
- 결제 방식 전환은 환경변수의 우연한 존재가 아니라 서버가 관리하는 명시적 기능 설정으로만 가능해야 한다.
- 금액, 재고 잠금, 입금 완료, 보관 권리 생성은 서버와 데이터베이스가 결정한다.
- 고객은 계좌번호와 `입금 대기 중` 상태를 확인한다. 권한 있는 A/B 운영자는 공용 대기열에서 입금자·관찰한 원장 버전을 확인하고, 서버가 잔액 영수증 추가와 결제 확정을 하나의 원자 작업으로 처리한다.
- 입금 확인은 중복 호출에 안전해야 하며 확인자, 시각, 금액, 입금자명, 취소·정정 사유를 감사 가능하게 남긴다. 부분·초과 입금과 원장 정정은 단순 확인 버튼과 분리한다.

## 주문 및 물류 불변식

- 한 체크아웃은 하나의 통합 주문과 하나의 수동 입금 요청을 만든다. 경매·차순위 구매도 동일한 수동 입금 확인 원칙을 사용한다.
- 매장별 업무 단위는 결제나 고객 주문을 분할하지 않는다. 상품 매출은 원등록 매장에 귀속하고 배송비는 중앙 사업체 수익으로 분리한다.
- 입금 확정은 상품별 구매자 보관 권리를 정확히 한 번 생성한다. 구매자는 결제 직후 `보관 준비 중`을 볼 수 있지만, 이는 B 입고 완료를 뜻하지 않는다.
- 배송 요청은 선택한 상품 단위의 불변 manifest다. 여러 주문·구매 유형을 합칠 수 있고, 같은 주문의 남은 상품은 이후 별도 배송 요청으로 보낼 수 있다.
- 활성 상품이 모두 B에 보관되고 각 원등록 매장 작업이 완료되며 배송비가 확정되고 미해결 예외가 없을 때만 포장과 송장을 허용한다. 하나라도 빠지면 `미 출고된 상품이 존재합니다`로 차단한다.
- 검수 필요, 분실, 오프라인 판매, 추가 대기는 상품별 예외 사건으로 기록한다. 사건은 재개, 다음 배송으로 제외, 또는 Owner 승인 수동 환불로만 해결하며 상품을 조용히 삭제하지 않는다.
- 상품별 위치와 책임 구간은 추적하되 고객에게는 선택 가능한 보관 상품과 배송 요청의 진행 상황을 보여 준다.

## 설계 판단 규칙
